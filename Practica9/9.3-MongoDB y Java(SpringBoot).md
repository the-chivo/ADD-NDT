# Tema9.3 MongoDB Atlas y Java (SpringBoot)

## 1. Creacion de un proyecto SpringBoot + MongoDB
* Ir a https://start.spring.io/
* Añadir la dependencia de Spring Data MongoDB

## 2. Creación de usuario en MongoDB atlas

* Registrarse en https://account.mongodb.com/account/login
* En el proceso de creación de un uruasrop se asignara un cluster para la DB
* Ademas se nos otorgara un string de conexion para conectar la base de datos con nuestra aplicacion.

## 3. Conexion con MongoDB Atlas

#### El string de conexion tiene la siguiente estructura:
mongodb+srv://<usuario>:<contraseña>@<cluster>.mongodb.net/<nombreBaseDatos>?retryWrites=true&w=majority
#### Debemos añadir esto a la propiedad "spring.data.mongodb.uri" del archivo Application.properties de SpringBoot
#### Al lanzar la aplicacion ya estaremos concetados a nuestra base de datos en la nube MongoDB Atlas

## 4. Anotacion @Document 
#### Esta anotacion indica que la clase representa un documento que vamos a persistir en la base de datos
#### Es similiar a @Entity utilizado para bases de datos relacionales
```
@Document
class User{
}
```
#### Definimos un modelo de datos para usuarios (una coleccion de documentos llamada "usuarios)
```
@Document(collection = "usuarios")
class User{
}
```

  ## 4.1 Anotacion @Id

  #### Indica que atributo es la clave primaria
  #### Es la misma anotacion utilizada para bases de datos relacionales

  ```
@Id
private String id;
```
####  Definimos un String que es clave primaria. En MongoDB Atlas se generan automaticamente sin idicar el modo de generacion

## 4.2 Anotacion @Field

#### Indica que el nombre con el que queremos que MongoDB guarde el campo del documento

```
@Document class User{
   String emailAddress
}
```

#### El campo emailAddress se guardara como "emailAddress"
```
@Document class User{
   @Field("email")
   String emailAddress
}
```
#### El campo emailAddress se guardara como "email"

## 4.3 Reglas de las clases documento

* Los atributos deben ser privados
* Los atributos deben tener getter y setters
* Debemos trener un constructor con parametros
* Debemos tener un constructoer sin parametros
* Tener un campo marcado como @Id es opcional(a diferencia de las entidades donde era obligatorio)

## 5 Repositorios: MongoRepository
Es una interfaz que proporciona metodos de acceso a la base de datos MongoDB para realizar operaciones CRUD(crear, leer, actualizar, eliminar) de manera sencilla. Esta diseñada para simplificar la interaccion con MongoDB y eliminar la necesidad de escibir consultas manuales.
#### User.java:
```
@Document class User{
   String emailAddress:

   public User(String email){
       this.emailAddress = email;
   }
}
```
#### UserRepository.java:
```
public interface UserRepository extends MongoRepository<User, String>{}
```
Para crear un objeto de tipo UserRepository haríamos:
#### Clase main de la aplicacion:
```
@Autowired
private UserRepository userRepository;
```
Al heredar de MongoRepository obtenemos una gran cantidad de metodos
CRUD ya implementados por defecto:
#### Metodos CrudRepository:
count, delete, deleteAll, deleteAllById, existsById, findById, save
#### Metodos ListCrudRepository:
findAll, findAllById, saveAll
#### Ejemplo
```
Long totalEntitiesInDatabase = userRepository.count();
userRepository.deleteAll();
User user = userRepository.findById(“67714fba07fef37a0aa50e51”);
```
Todos los metodos: https://docs.spring.io/spring-data/mongodb/docs/current/api/org/springframework/data/mongodb/repository/MongoRepository.html.

A pesar de esta gran cantidad de métodos CRUD incluidos por defecto, podemos añadir nuestros metodos personalizados utilizando una serie de "keywords"

#### User.java:
```
@Document class User{
   String emailAddress;
   int age
   /...
}
```
#### UserRepository.java
```
public interface UserRepository extends MongoRepository<User, String>{
   List<User> findByAgeGreaterThan(int age);
   List<User> findByAgeBetween(int min, int max);
   Void deleteByEmailAddressContaining(String keyword);
}
```
